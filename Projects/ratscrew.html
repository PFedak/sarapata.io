About

Language ~ Java

This coding project was the first large coding project I ever worked on. This was coded in early 2019 and has barely been changed since then, except for re-working the code to work on a new computer.

This code is definitely still has some bugs and the interface looks something out of the mid 2000's, but I wanted to showcase real early work projects. This project was the first time I had ever worked with a project that does not only print output to the screen.

The program took many hours of meticulous work, not only to create a layout for the first time, but also implement rules to a real world game. The output is something I'm proud of for a first real programming attempt.

Rules

Dealing

Deal cards one at a time to each player until all cards have been dealt, ensuring to keep the cards face down.

Playing

The player to the left of the dealer begins. To play, pull the top card off of your un-visible hand and place it face up in a pile on the middle of the table. This continues until someone puts down a face card. If the next player does not put down another face card within their alloted chande, the person who placed the last face card gets the pile. The only thing that overrides the face card rule is a slap.

The Slap

Double ~ When two cards of equivelent value are placed consecutively.

Sandwich ~ When two cards of equivelent value are placed consecuitvely with only one card in between them.

If there was an invalid slap, you must add a card to the bottom of the pile.

Winning

To win, one player must pick up all the cards.

Interface

Computer/Player:

The computer or player components each have a layered pane and a label. The layed pane holds the images of their cards, has an outline, and is titled according to which hand it is.

Pile:

The pile is very similar to the computer or player layout, except it also has a label for important events such as: who picked up the pile, if someone slapped, etc.

Buttons:

The buttons are placed in their own panel. There are buttons to slap, place a player card, place a computer card, or start a new game. A button can only be pressed to play a card if it is that players turn.

Code

Code Class

The Card class is only intended to keep track of one singular card. It has four private variables: rank, suit, value, and image. There are two constuctions. One which creates an empty card and a second which fills in all the variables. There are also get functions for each of the variables and a toString method to return a string value of "rank of the suit".

Hand Class

The Hand class is intended to keep track of only one players hand (computer, player, or pile). It has two private variables, one for the number of cards and one ArrayList of cards. Hand has two constructors. One of which to create an empty hand, and the other to create a hand of a list of cards. This class has the methods sets and gets for both variables, adding one card, adding a card to a certain position, removing one card based off of position, getting the card at a certain position, clearing the array, and adding or remove one value from the size.

Deck Class

The Deck class keeps track of the deck. It has three private variables: an array of the deck, the index of the next card to be dealt, a final variable for number of cards, and a String array of the files for the card images. Deck only has one constructor, which creates a Card for the amount of cards to be dealt. Deck also has the methods to: shuffle, and a card.

RatscrewGUI Class

The RatscrewGUI class is where the interface is created and the rules of the game are inforced. The class has various private variables which are for the panels, labels, layered panes, buttons, deck, cards, hands, slaps, and recent cards we need. It only has one constructor which builds the panel.

Methods

The build panel method initializes all of the private variables.
The deal hand method deals cards to the hand passed to it.
The reprint hand method will recreate the images of the back of the cards that are pictured in the computer or player hands.
The start game method resets all positions, deals cards to each player, and forces the computer to place a card.
The put card method allows either hand to place a card on top of the pile. pickPile allows either hand to pick up the enitre pile.
placeCards determines of either player needs to pick up the pile.
GameOver determines if the game is over.
Slap determines if a slap is valid.
ComputerSlap will randomly decide to slap based off of a random number generator.

The final portion of the code handles the button's actions.

CardApp Class

The only purpose of the CardApp class is to create the frame, set the size, make the frame visible, and close the frame at the end.
