Language ~ Java

Lang is a programming language that was created for a class project. Lang operates a parser and interpreter and can handle mathematic operations, if statements, while loops, functions and classes, and variable types. Lang keeps track of parses using trees. The program has 13 classes: Parse, Parser, IntegerParse, StatementParse, VariableParse, Value, IntegerValue, ClassValue, FunctionValue, Environment, Interpreter, Location, and Main.

Code

Parse Class

The Parse class has three protected variables, the name of the parse node, the index of the last character we parsed through, and a LinkedList of the children. It only has one constructor which initalizes name and index. There is a method to see if two parses are equal as well as normal sets and gets.

Parse Subclasses

The StatementParse and IntegerParse classes have a boolean variable to keep track if the variable is typed, as this is optional. The StatementParse class has a protected variable of a LinkedList for children. While the VariableParse class has a VariableParse variable. Both classes have the similare constructors and methods as Parse, but they also hav a toString which is used for keeping track of the tree.

Parser Class

The Parser parses through the statements and makes sure that the correct syntax is followed. Since there is a lot of logic in this class, it is quite large. However, this class was the most straightforward to code because it was all enforcing syntax.

Value Class

The Value class has one protected variable to keep track of its name. Its only constructor initalizes name. The class can return the name or get the integer value. Value can also get the environment, indetifier, object, parameters, types, or return type. It can also add, subtract, multiply, or divide the value. Value also has an equal method using both a Value and an int as well as not equals, less than or equal to, greater than or equal to, less than, and greater than for both parameter types.

Value Subclasses

These classes are all subclasses of Value. IntegerValue has an int to keep track of the value and also has a constuctor to initalize value, get the value, and return a toString. FunctionValue also has a variable to keep track of the intermediate representation of the function, the environment, the parameters, the types, the return types, and a boolean if it has a signature. It only has a constructor and a toString. ClassValue has an itermediate representation as well as the environment and has similar methods to FunctionValue. The Location class has the variables for where the value is store and its indetifier. It only has a constructor.

Environment Class

The Environment class keeps track of the environment at any given moment. This class is important for function and class calls.

Interpreter Class

The Interpreter class is where lang actually interprets the program. It is the class that actually evaluates the mathematical properties ad well as the statements lang can handle, classes and functions. Interpreter has a method to evaluate any statement that lang can parse.

Main Class

The Main class runs lang against set test cases to insure that the language is running correctly. It can test using files and directories as well as giving an error message.
